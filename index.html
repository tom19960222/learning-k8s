這個網站涵蓋什麼？

從原始碼出發深入分析 5 個 Kubernetes 生態系專案，並附一份按時間軸推進的 30 天 hands-on lab。

Kubernetes 核心

k8s 控制平面、kubelet、kube-proxy、scheduler 與其 CRD / API server 設計。

網路與儲存

cilium 的 eBPF datapath、multus 多網路、ceph 分散式儲存。

虛擬化平台

KubeVirt 的 VMI 生命週期、virt-controller / handler / launcher 三層架構。

學習路徑建議

🟢 初學者
剛接觸 Kubernetes 生態系
  1. ① 從 30 天 lab 開始
    依時間軸動手做實驗，快速建立可操作的整體圖像
  2. ② 配合各專案 overview
    每天 lab 完成後，閱讀對應專案的概觀頁建立背景知識
  3. ③ 完成互動測驗
    以選擇題自我檢查，找出仍模糊的概念回頭補強
🟡 中階工程師
已會基本操作，要懂內部運作
  1. ① 由 architecture 切入
    聚焦各專案的元件職責與資料流，看清誰跟誰互動
  2. ② 跟著 reconcile loop 走
    挑一個 Controller，從事件進入到 status 寫回的完整路徑
  3. ③ 對照 lab 結果驗證
    用 kubectl describe / events 對齊原始碼上的狀態變化
🔴 資深工程師
要深度貢獻或建構平台
  1. ① 跨專案閱讀
    同時看 cilium / multus / kubevirt 的 CNI 介接，比較設計取捨
  2. ② 邊界條件分析
    研究錯誤路徑、finalizer、leader election 的失敗模式
  3. ③ 自行擴充
    依 lab 末段挑戰題目實作小型 controller / CNI 外掛 / VM 操作工具
