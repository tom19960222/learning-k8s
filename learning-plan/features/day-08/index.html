觀念深化

cilium-agent 是 DaemonSet，每個 node 一個 Pod。它的職責是：編譯 eBPF 程式（C source 位於 bpf/ 目錄）、透過 BPF verifier 載入 kernel、維護 BPF map（存放 endpoint metadata、policy rules、service backends），以及管理 network namespace 的 veth pair。每個 Pod endpoint 在 ingress / egress 方向各有一個 eBPF program。

cilium-operator 是 Deployment（通常只需要一個 leader 副本）。它負責：分配 node CIDR（當 ipam.mode=cluster-pool 時）、把 K8s Service 轉成 cilium 內部 service objects、垃圾回收 orphaned endpoint 等跨 node 的控制平面工作。

換 CNI 的關鍵要點：CNI plugin binary 放在 node 的 /opt/cni/bin/，設定檔放在 /etc/cni/net.d/（字典序最小的檔案先被 kubelet 使用）。cilium 安裝完後會在每個 node 寫入 /etc/cni/net.d/05-cilium.conflist，接管新建立的 Pod 網路。
