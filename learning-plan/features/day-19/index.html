觀念深化

CephFS 是 POSIX 檔案系統 over RADOS。資料還是落 RADOS object（在 data pool），metadata（dir entry、inode、檔名、權限、locks）由 MDS 服務。MDS 是 stateful daemon —— active MDS 在 memory 維護一個 metadata cache，讀寫都先過它。

為什麼 RBD 不需要 MDS、CephFS 需要？因為 RBD 是 single-client (single-Pod) 直接讀寫 object，不需要協調；CephFS 是 multi-client filesystem semantics，需要 MDS 集中協調 inode lock、處理 capability lease。

CephFS 也能 scale-out：在 active MDS 上 ceph fs set k8s-fs max_mds 2 可以加成 multi-active，metadata namespace 會被 partition 到不同 MDS（subtree partitioning）。
