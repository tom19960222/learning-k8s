觀念深化

ceph-csi 由兩部分組成：

  • controller plugin（Deployment） — 跑在某個 node 上，負責「建立 / 刪除 volume」「擴容」等需要呼叫 ceph API 的操作。對應 PVC 的生命週期。
  • node plugin（DaemonSet） — 每個 node 一個，負責「stage / publish 到 Pod」「unstage / unpublish」。當 Pod 排到該 node 才需要這個 plugin。

兩者都實作 CSI gRPC interface（CSI spec）。kubelet 不需要懂 ceph，只跟 CSI socket 對話。所以同一個 k8s cluster 可以同時跑很多 CSI driver（NFS、AWS EBS、ceph-rbd、ceph-cephfs 各自獨立）。

更深入：

  • 整個 controller-plugin 進程結構 internal/csi-common/
  • RBD provisioner 邏輯 internal/rbd/controllerserver.go 裡的 CreateVolume()

收穫檢查

  • 為什麼 controller plugin 要一個 secret，node plugin 也要一個？兩者用途差在哪？
  • 換成 SSD 專用的 pool 應該改 StorageClass 哪一欄？
