6. 看 launcher Pod 的 NIC 拓樸

你會看到：

  • eth0 — Pod 主介面（cilium，給 VM 透過 masquerade 用）
  • net1 — multus 加的介面，bridge to vmbr0
  • tap0 / tap1 — qemu 給每個 VM NIC 配的 tap 介面（VM 內看到的 eth0/eth1 對應 launcher namespace 的 tap）

觀念深化

KubeVirt VM 的網路繞了好幾圈：

VM 內 eth0
  ↓ qemu virtio
launcher Pod 的 tap0
  ↕ Linux bridge 或 NAT
launcher Pod 的 eth0 (CNI 給的)
  ↓ veth pair
node 的 host-side veth
  ↓
cilium eBPF datapath / iptables
  ↓
node eth0 → 物理網路

第二張 NIC（multus bridge）就比較直：

VM 內 eth1
  ↓ qemu virtio
launcher Pod 的 tap1
  ↕
launcher Pod 的 net1 (multus 加的)
  ↓ veth pair 進 node 的 vmbr0
vmbr0 物理 bridge → 物理網路

注意：第二張網卡走 multus 後，cilium 的 NetworkPolicy 不適用這條路徑。安全策略要靠 bridge 上層的 firewall（例如 Proxmox 端 firewall 或 nftables）。
