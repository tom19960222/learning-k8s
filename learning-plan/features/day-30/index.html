30 天回顧

從 Day 1 第一次 ssh 進 control plane node、發現 kube-apiserver 是個 static Pod，到 Day 30 親手部署一個跨儲存、網路、虛擬化的 web service VM —— 你走過了一整段 k8s 生態系的學習路徑。

Block 1 (Day 1-7) 把 k8s 的 4 個控制平面元件 + kubelet + kube-proxy 拆解，最後親手寫一個 controller — 從此 Reconcile() 不再是黑魔法。

Block 2 (Day 8-14) 把網路層換成 cilium，看到 eBPF 真實在跑；補上 multus 為 VM 的多網路鋪路。

Block 3 (Day 15-21) 進入儲存層，先理解 Proxmox-managed ceph 的 RADOS / CRUSH / PG 模型，再用 ceph-csi 把它接進 k8s；同時花一個 weekend 看 Rook operator 怎麼用 k8s pattern 管 ceph。

Block 4 (Day 22-30) 終於把 KubeVirt 裝起來，從第一台 fedora VM 一路堆到 PVC 磁碟 → 多網路 → live migration → 整合 demo。每一步都連回前面 block 累積的東西。

接下來可以做的：

  • 把這些原始碼的細節寫進站內的 5 個專案分析頁（SP-3..SP-7）—— 對應的子計畫已在 docs/superpowers/specs/ 規劃中
  • 把 demo VM 包成 Helm chart 或 GitOps workflow（ArgoCD / FluxCD）
  • 用 Cluster API Provider KubeVirt 把整個 k8s cluster 當作「VM 平台」管理，做巢狀 cluster
  • 加上 monitoring stack（kube-prometheus-stack）、log（Loki）、tracing（Jaeger）

謝謝你陪這個 30 天計畫走到這裡。

收穫檢查
