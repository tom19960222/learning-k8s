## 動手做

### 1. 新增 Cilium Helm repo

```bash
# 🟡 已對照官方文件 / 原始碼
helm repo add cilium https://helm.cilium.io/
helm repo update
```

```
"cilium" has been added to your repositories
Hang tight while we grab the latest from your chart repositories...
...Successfully got an update from the "cilium" chart repository
```

### 2. 確認 cluster CIDR

安裝前先確認你的 cluster 使用什麼 Pod CIDR，稍後 `--set` 參數要一致：

```bash
# 🟡 已對照官方文件 / 原始碼
kubectl cluster-info dump | grep -i cluster-cidr | head -1
```

```
"--cluster-cidr=10.244.0.0/16",
```

記下這個 CIDR（預設通常是 `10.244.0.0/16` 或 `192.168.0.0/16`，依 cluster 建立方式而定）。

### 3. 移除既有 CNI（選做）

若你的 cluster 已有 flannel 或 calico，移除它會中斷現有 Pod 網路。**如果你的學習 cluster 可以接受短暫中斷，才執行這步；若不確定請跳過此步，直接安裝 cilium（有些環境可以共存）：**

```bash
# 🔴 需在你環境跑後對照
# 以 flannel 為例（calico 請參考 calico 文件移除方式）
kubectl delete -f https://raw.githubusercontent.com/flannel-io/flannel/master/Documentation/kube-flannel.yml
```

移除後等待舊的 flannel Pod 消失：

```bash
# 🟢 已驗證 (kind / 顯然正確)
kubectl -n kube-flannel get pods
```

### 4. 安裝 cilium v1.19.3

```bash
# 🟡 已對照官方文件 / 原始碼
helm install cilium cilium/cilium \
  --version 1.19.3 \
  --namespace kube-system \
  --set kubeProxyReplacement=false \
  --set ipam.mode=kubernetes
```

```
NAME: cilium
LAST DEPLOYED: ...
NAMESPACE: kube-system
STATUS: deployed
REVISION: 1
```

`kubeProxyReplacement=false` 表示繼續讓 kube-proxy 處理 Service，cilium 只做 Pod 網路。`ipam.mode=kubernetes` 讓 cilium 沿用 cluster 原本配置的 Pod CIDR，不自行管理 IP 池。

### 5. 等待所有 cilium-agent 就緒

```bash
# 🟢 已驗證 (kind / 顯然正確)
kubectl -n kube-system get pods -l k8s-app=cilium -w
```

```
NAME                READY   STATUS    RESTARTS   AGE
cilium-xxxxx        1/1     Running   0          2m
cilium-yyyyy        1/1     Running   0          2m
```

當所有 Pod 都顯示 `1/1 Running` 之後，按 `Ctrl+C` 結束 watch。

### 6. 確認 cilium-operator 健康

```bash
# 🟢 已驗證 (kind / 顯然正確)
kubectl -n kube-system get deploy cilium-operator
```

```
NAME              READY   UP-TO-DATE   AVAILABLE   AGE
cilium-operator   1/1     1            1           3m
```

## 觀念深化

**cilium-agent** 是 DaemonSet，每個 node 一個 Pod。它的職責是：編譯 eBPF 程式（C source 位於 `bpf/` 目錄）、透過 BPF verifier 載入 kernel、維護 BPF map（存放 endpoint metadata、policy rules、service backends），以及管理 network namespace 的 veth pair。每個 Pod endpoint 在 ingress / egress 方向各有一個 eBPF program。

**cilium-operator** 是 Deployment（通常只需要一個 leader 副本）。它負責：分配 node CIDR（當 ipam.mode=cluster-pool 時）、把 K8s Service 轉成 cilium 內部 service objects、垃圾回收 orphaned endpoint 等跨 node 的控制平面工作。

換 CNI 的關鍵要點：CNI plugin binary 放在 node 的 `/opt/cni/bin/`，設定檔放在 `/etc/cni/net.d/`（字典序最小的檔案先被 kubelet 使用）。cilium 安裝完後會在每個 node 寫入 `/etc/cni/net.d/05-cilium.conflist`，接管新建立的 Pod 網路。
