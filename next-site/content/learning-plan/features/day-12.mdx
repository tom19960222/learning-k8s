## 動手做

### 1. Helm upgrade：啟用 WireGuard encryption

```bash
# 🟡 已對照官方文件 / 原始碼
helm upgrade cilium cilium/cilium \
  --version 1.19.3 \
  --namespace kube-system \
  --reuse-values \
  --set encryption.enabled=true \
  --set encryption.type=wireguard
```

```
Release "cilium" has been upgraded. Happy Helming!
STATUS: deployed
REVISION: 3
```

### 2. 等待 DaemonSet rollout

```bash
# 🟢 已驗證 (kind / 顯然正確)
kubectl -n kube-system rollout status ds/cilium
```

```
daemon set "cilium" successfully rolled out
```

每個 cilium-agent Pod 重啟後會自動產生 WireGuard keypair，並透過 K8s API（CiliumNode object）交換公鑰。

### 3. 確認加密狀態

```bash
# 🟡 已對照官方文件 / 原始碼
cilium status | grep -i encrypt
```

```
Encryption:         Wireguard   [cilium_wg0 (Pubkey: <base64-key>, Port: 51871, Peers: 2)]
```

`Peers: 2` 對應兩個 node（control-plane + worker；或依你的 cluster node 數）。

### 4. 在 node 上查看 WireGuard 介面

SSH 進任一 node：

```bash
# 🟡 已對照官方文件 / 原始碼
wg show
```

```
interface: cilium_wg0
  public key: <this-node-pubkey>
  private key: (hidden)
  listening port: 51871

peer: <other-node-pubkey>
  endpoint: <other-node-ip>:51871
  allowed ips: 0.0.0.0/0, ::/0
  latest handshake: X seconds ago
  transfer: X MiB received, X MiB sent
```

`cilium_wg0` 是 cilium 自動建立與管理的 WireGuard 介面。`peer` 清單列出所有其他 node 的公鑰與 endpoint。`latest handshake` 若在幾秒內表示 tunnel 正常運作。

### 5. 抓取加密封包

找到 node 的對外網路介面名稱（例如 `eth0`、`ens3`、`enp3s0`）：

```bash
# 🟡 已對照官方文件 / 原始碼
ip route get 8.8.8.8 | awk '{print $5; exit}'
```

用 `tcpdump` 抓 WireGuard UDP 封包：

```bash
# 🟡 已對照官方文件 / 原始碼
tcpdump -i <node-iface> -nn 'udp port 51871' -c 5
```

```
13:42:01 IP 192.168.1.101.51871 > 192.168.1.102.51871: UDP, length 148
13:42:01 IP 192.168.1.102.51871 > 192.168.1.101.51871: UDP, length 148
...
```

這些 UDP 封包就是加密後的 Pod 間流量。payload 是 WireGuard 加密的 ChaCha20-Poly1305 密文，wireshark 也看不出原始 Pod IP。

### 6. 製造跨 node 流量並確認加密

在另一個 terminal 先確保有跨 node 的 Pod：

```bash
# 🟡 已對照官方文件 / 原始碼
kubectl run cross-test --image=curlimages/curl -- sleep 600
kubectl get pod cross-test -o wide
```

若 Pod 恰好在同一個 node，刪除重跑直到排到不同 node。確認跨 node 後：

```bash
# 🟢 已驗證 (kind / 顯然正確)
# 在 cross-test Pod 裡 curl 另一個 node 上的 Pod
kubectl exec cross-test -- wget -q -O - http://10.244.x.x/  # 對端 node 上的任一 Pod IP
```

同時在 `tcpdump` terminal 應能看到對應時間點的 UDP/51871 封包，確認流量確實走 WireGuard 加密通道。

### 7. 清理

```bash
# 🟢 已驗證 (kind / 顯然正確)
kubectl delete pod cross-test
```

## 觀念深化

**Cilium WireGuard transparent encryption** 的設計目標是「對 Pod 完全透明」——Pod 不知道底層有 WireGuard，它還是對普通 Pod IP 發送封包。Cilium 在 node 上的 eBPF 路由邏輯把跨 node 的封包導向 `cilium_wg0` 介面，WireGuard kernel module 把封包加密後以 UDP 送出；對端 node 收到 UDP 後 WireGuard 解密，再交還給 eBPF routing 送到目標 Pod。

與 IPsec（cilium 也支援）比較：WireGuard 的 handshake 更簡單（基於 Diffie-Hellman，沒有 IKE 協商複雜度），且完全在 kernel 實作（Linux 5.6+ 原生支援），效能優異、實作更少複雜度。

**公鑰交換機制**：每個 cilium-agent 把自己的 WireGuard 公鑰存到 `CiliumNode` CR 的 annotations（`network.cilium.io/wg-pub-key`）。其他 node 的 agent watch 這個欄位，自動把 peer 的公鑰加進 WireGuard 設定。這樣新加 node 只需要 K8s API 就能完成金鑰交換，不需要額外的 key management infrastructure。

參考：
- [Cilium WireGuard Encryption](https://docs.cilium.io/en/v1.19/security/network/encryption-wireguard/)
