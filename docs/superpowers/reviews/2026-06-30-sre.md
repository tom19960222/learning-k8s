# SRE agent review — ceph-incident-bundle (2026-06-30)

Production-operations lens. No cluster mutations found (read-only verdict confirmed).

## HIGH
- `run/collect.sh:166-169` — single `--timeout` (20s) bounds the WHOLE-node SSH wrapper while the remote runs ~25 sequential commands → any node >20s total is killed and recorded as total failure; drops the most valuable (slow/large) node almost every time. Fix: split per-command timeout from a much larger `--node-timeout` (default ~600s) used only for the outer wrapper.
- `run/collect.sh:284-330` — no `trap` anywhere; Ctrl-C / mid-run failure leaves `tmp.<ts>.$$` workdir + in-flight `.node-*.tar.gz` on the workstation, accumulating across runs. Fix: `trap cleanup EXIT INT TERM` installed right after workdir computed, honoring `--keep-workdir`.
- `run/collect.sh:171-187` — `tar | ssh` rc under pipefail + a timeout-killed/partial SSH can still leave a non-empty extractable `$node_tar` → node counted `node_ok` with silently truncated contents. Fix: after extraction assert node produced its own `manifest.jsonl`; absent → SKIPPED + rc=2. Detect rc==124 distinctly.
- `lib/collect-node.sh:296,299` — `dmesg -T` and `journalctl --since -u 'ceph*'` bounded by the small per-command timeout; large journals exceed 20s → SIGTERM → silently truncated artifact that looks complete. Fix: larger dedicated timeout (or `--lines`/`--until`), and on rc==124 append `# TRUNCATED: timed out` trailer.

## MED
- `lib/common.sh:105-109` — `run_capture` can't distinguish rc=1 (ran+failed) from rc=124 (timed out); manifest shows generic non-zero. Fix: detect 124, set `"timed_out":true`, write `# TIMEOUT` marker, surface count in summary.txt.
- `lib/collect-node.sh:169` — logs > 1MiB cap silently skipped, no marker; the active large osd log is exactly what's wanted. Fix: copy `tail -c cap` and/or write `SKIPPED-oversize.txt` listing path+size.
- `run/collect.sh:317,321` — `verify-bundle.sh` under `set -e`: the dir-verify at :317 aborts main BEFORE the tar is produced → collected evidence destroyed when verify flags something. Fix: run verify with `set +e`, still produce `.tar.gz`, record failure, exit 1; never discard evidence.
- `lib/collect-cluster-cephadm.sh:18-27` — cluster ceph SSH has no `ConnectTimeout`/`ServerAlive*`; half-open seed (partial outage) hangs; wedged `cephadm shell` (quorum lost) burns full timeout × ~24 commands serially → multi-minute stall. Fix: add ConnectTimeout/ServerAliveInterval/ServerAliveCountMax.
- `run/collect.sh:299-312` — malformed HOSTS entry calls `die` mid-loop, aborting after some nodes collected, discarding partial bundle. Fix: append_error + node_failed++ + continue + rc=2.
- `lib/collect-node.sh:60` — `sudo -n` denied flips an otherwise-good node to failed (rc=2). Fix: treat sudo-denied as recorded skip, not hard node failure.

## LOW
- `lib/common.sh:51-80` / `verify-bundle.sh:24-30` — line-based redaction on fixed extension allowlist; base64 keyrings / inline tokens pass through. Fix: add code comment documenting intentional gaps (real scanning = feature, out-of-scope).
- `lib/common.sh:58,97` — `mktemp`/`mv` into artifact dir; full/RO fs makes `set -e` abort whole run. Fix: guard with error handling, record + continue.
- `run/collect.sh:160-164` — remote `${TMPDIR:-/tmp}` full/RO → empty/garbage tar → generic SKIPPED with no root cause. Fix: test writability, emit specific `SKIPPED: remote tmp not writable`.
- `run/collect.sh:284` — workdir under repo `results/`; no pre-flight free-space check. Fix: cheap `df` warn (not abort).
- `lib/collect-cluster-rook.sh:116,127` — `kubectl logs --since` / `exec ceph status` inherit small timeout → truncation. Fix: same per-command-vs-bulk split, mark truncation.
