5 еҖӢе…ғд»¶зҡ„иҒ·иІ¬йӮҠз•Ң

kube-apiserver жҳҜе”ҜдёҖеҸҜд»Ҙи®ҖеҜ« etcd зҡ„ processгҖӮжүҖжңүе…¶д»–е…ғд»¶ (еҗ« kubeletгҖҒcontrollersгҖҒuser з«Ҝ kubectl) йғҪйҖҸйҒҺе®ғзҡ„ HTTPS API е°Қ cluster зӢҖж…ӢеҒҡ CRUDгҖӮе®ғжҳҜж•ҙеҖӢ cluster зҡ„ source of truth proxyгҖӮйҖІе…Ҙй»һпјҡ

etcd жҳҜеҲҶж•ЈејҸ KV store (Raft consensus)пјҢеӯҳ cluster е…ЁйғЁзӢҖж…Ӣ (PodгҖҒNodeгҖҒSecret зӯү)гҖӮе®ғеҸӘжҺҘ apiserver зҡ„йҖЈз·ҡвҖ”вҖ”жІ’жңүеҲҘзҡ„ clientгҖӮжһ¶ж§Ӣжұәе®ҡе®ғжІ’жңү application-aware зҡ„ schema validationпјӣйӮЈжҳҜ apiserver зҡ„е·ҘдҪңгҖӮ

kube-controller-manager жҳҜ 1 еҖӢ process и·‘ ~30 еҖӢ controller (ReplicaSetгҖҒDeploymentгҖҒJobгҖҒEndpointвҖҰ)пјҢжҜҸеҖӢ controller жҳҜдёҖеҖӢ goroutineгҖӮе®ғеҖ‘ watch apiserverгҖҒи·‘ reconcile loopгҖҒеҜ«еӣһгҖӮи©іиҰӢ controllersгҖӮ

kube-scheduler д№ҹжҳҜзҚЁз«Ӣ processгҖӮе®ғеҸӘеҒҡдёҖд»¶дәӢпјҡwatch spec.nodeName == "" зҡ„ PodпјҢи·‘ predicate + score з®—жі•йҒё nodeпјҢе‘јеҸ« apiserver зҡ„ Bind еҜ«еӣһ spec.nodeNameгҖӮеҫҢзәҢе°ұжҳҜ kubelet жҺҘжүӢгҖӮ

kubelet жҳҜжҜҸеҖӢ node дёҠзҡ„ daemon (еҗ« control plane node иҮӘе·ұ)гҖӮе®ғ watch иҮӘе·ұ node дёҠиҰҒи·‘зҡ„ PodгҖҒйҖҸйҒҺ CRI gRPC и·ҹ container runtime е°Қи©ұгҖҒе‘јеҸ« CNI plugin й…Қз¶Іи·ҜгҖӮи©іиҰӢ kubeletгҖӮ

kube-proxy жҳҜ per-node DaemonSet (technically static Pod д№ҹиЎҢ)пјҢwatch Service + EndpointSliceпјҢжҠҠ cluster IP вҶ’ backend Pod зҡ„е°ҚжҮүеҜ«жҲҗ iptables / IPVS / nftables иҰҸеүҮгҖӮжІ’жңү proxy process еңЁ data pathвҖ”вҖ”kernel netfilter зӣҙжҺҘеҒҡ DNATгҖӮ

kubectl apply зҡ„ж—…зЁӢ

з•¶дҪ  kubectl apply -f deployment.yamlпјҡ

  1. kubectl и§Јжһҗ yamlгҖҒиө° ~/.kube/config жӢҝ cluster URL иҲҮ token
  2. е°Қ https://<apiserver>/apis/apps/v1/namespaces/default/deployments зҷј POST
  3. apiserver ж”¶еҲ° вҶ’ admission chain (mutating вҶ’ validating) вҶ’ schema validation вҶ’ еҜ« etcd вҶ’ еӣһ 200
  4. kube-controller-manager иЈЎзҡ„ Deployment controller watch еҲ°ж–° Deployment вҶ’ е»ә ReplicaSet вҶ’ еҜ« apiserver
  5. ReplicaSet controller watch еҲ°ж–° ReplicaSet вҶ’ е»ә N еҖӢ Pod (spec.nodeName == "") вҶ’ еҜ« apiserver
  6. kube-scheduler watch еҲ° unscheduled Pod вҶ’ и·‘з®—жі• вҶ’ е°Қ apiserver зҷј Bind еҜ« spec.nodeName=worker-X
  7. worker-X зҡ„ kubelet watch еҲ°иҮӘе·ұ node еӨҡдәҶдёҖеҖӢ Pod вҶ’ жӢү image вҶ’ йҖҸйҒҺ CRI иө· sandbox вҶ’ иө· application container вҶ’ е‘јеҸ« CNI й…Қз¶Іи·Ҝ
  8. kubelet жҢҒзәҢжҠҠ Pod status е ұеӣһ apiserverпјӣдҪ  kubectl get pod зңӢеҲ° Running

жіЁж„ҸжҜҸеҖӢе…ғд»¶еҸӘи·ҹ apiserver е°Қи©ұвҖ”вҖ”еҪјжӯӨдёҚзӣҙжҺҘйҖҡиЁҠгҖӮйҖҷжҳҜ k8s зҡ„ж ёеҝғи§ЈиҖҰиЁӯиЁҲгҖӮ

Static Podпјҡcontrol plane иҮӘе·ұжҖҺйәјиө·дҫҶ

йӣһз”ҹиӣӢиӣӢз”ҹйӣһзҡ„е•ҸйЎҢпјҡapiserver жҳҜ PodпјҢдҪҶе»ә Pod йңҖиҰҒ apiserverгҖӮи§Јжі•жҳҜ static Podпјҡkubelet е•ҹеӢ•жҷӮи®Җжң¬ж©ҹ /etc/kubernetes/manifests/*.yamlпјҢжҠҠжүҫеҲ°зҡ„ yaml зӣҙжҺҘз•¶ Pod иө·пјҢдёҚз¶“йҒҺ apiserverгҖӮ

control plane node дёҠиЈқ kubeadm жҷӮйҖҷзӣ®йҢ„жңғжңү 4 еҖӢ yamlпјҡ

kubelet иө·йҖҷ 4 еҖӢ Pod д№ӢеҫҢпјҢapiserver иө·дҫҶпјҢж•ҙеҘ— control plane е°ұеҸҜд»Ҙ self-host дәҶгҖӮйҖҷд№ҹжҳҜзӮәд»Җйәј kubectl get pods -n kube-system иғҪзңӢеҲ° kube-apiserver-<control-plane-node> йҖҷзЁ® PodвҖ”вҖ”е®ғзңҹзҡ„жҳҜ PodпјҢдҪҶжІ’жңү ReplicaSet з®Ўе®ғпјҢжҳҜ kubelet зӣҙжҺҘз®ЎгҖӮ
